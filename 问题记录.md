# 开发过程中遇到的问题及解决方案

## 问题1: 透明PNG丢失透明度
**现象**: 用 `selection.copy()` + `paste` 粘贴透明 PNG 后，印花变成黑色矩形。
**原因**: copy+paste 会丢失 alpha 通道。
**解决**: 改用 `ArtLayer.duplicate()` 图层复制，完整保留透明度。


## 问题2: JS 里 bounds 返回 UnitValue 对象
**现象**: 位置计算错误。
**原因**: `bounds[0]` 返回 UnitValue 对象，不是数值；未设置单位为像素。
**解决**:
```javascript
app.preferences.rulerUnits = Units.PIXELS;
var layerLeft = bounds[0].value;  // 用 .value 取数值
```


## 问题3: 最高有效像素计算被噪点干扰
**现象**: 位置有偏差。
**原因**: 直接取 `ys.min()` 会被顶部噪点干扰。
**解决**: 从上往下找，找到**一行有效像素≥50个**的那行才算（复刻美图秀秀逻辑）。


## 问题4: 中心点x用简单平均值不准确
**现象**: X方向位置偏差。
**原因**: 用 `xs.mean()` 简单平均值。
**解决**: 用透明度加权质心 `(x * alpha).sum() / alpha.sum()`。


## 问题5: 缩放比例基准不一致
**现象**: 正图用 resize(10) 生成，但比手动"置入后设10%"小。
**原因**: 用户手动是"PS置入+设10%"，PS置入会自动缩放；duplicate方式resize(10)是相对原始尺寸。
**解决**: 实测手动"10%"=相对原始尺寸13.33%，正图用 `scale_percent=13.33`。


## 问题6: PS resize(TOPLEFT) 锚点bug（最难发现，最关键）
**现象**: 不同贴图位置偏差不同——DX0086正确(570)，DX0125偏下(748)。
**原因**: PS的 `resize(scalePercent, TOPLEFT)` 不是以图层(0,0)为锚点缩放，
而是以**有效像素区域左上角**为锚点。导致透明区域不缩放，bounds.T不正确：
- DX0086: 顶部透明125行，resize后bounds.T=2（接近0，PS偶尔正常）
- DX0125: 顶部透明276行，resize后bounds.T=272（未缩放，应该是82.8）

**调试过程**:
1. 发现translate移动量正确，但移动后bounds.T偏大
2. 对比duplicate后bounds.T vs 缩放后bounds.T，发现resize没缩放透明区域
3. 确认PS resize锚点以有效像素区域左上角为准

**最终解决**: **Python预先trim透明边距 + 缩放**，保存为临时PNG。
PS只需duplicate+移动+旋转，完全避开resize的bug。
trim后图层左上角=有效像素左上角，移动 moveY=targetTopY 即可对齐。
验证：DX0086 Y=568、DX0125 Y=567，都接近目标570。✅


## 问题7: 用错误的Python差异检测验证位置（教训）
**现象**: 用 `diff_sum > 10` 检测位置，错误地说"接近✅"。
**原因**: 旋转后边角半透明像素干扰，检测位置偏大。
**教训**: 不要用不准确的检测方法下结论。应该用alpha通道或JSX日志验证。


## 问题8: JSX 路径转义问题
**解决**: 用 `.replace("\\", "\\\\")` 显式替换。


## 问题9: BW 类型重复生成
**解决**: 重构为独立分支。
