# 开发过程中遇到的问题及解决方案

## 问题1: 设计图在输出中完全消失（旧版）
**现象**: 生成的 JPG 只有 T 恤模板，没有贴图。
**原因**: 最初误以为设计图是黑色背景，实际是**透明 alpha 通道**（alpha>=20 为有效像素）。

**最终方案**: 不需要转透明，设计图本身就是透明的。直接用即可。


## 问题2: JS 里粘贴透明PNG丢失透明度（关键问题）
**现象**: 用 `selection.copy()` + `paste` 粘贴透明 PNG 后，印花变成黑色矩形，盖住胚衣。
**原因**: `selection.selectAll()` + `copy` + `paste` 的方式会丢失 alpha 通道，透明区域变成黑色。

**最终方案**: 改用 `ArtLayer.duplicate()` 图层复制方式，完整保留透明度：
```javascript
var designLayer = designDoc.artLayers[0].duplicate(doc);
designDoc.close(SaveOptions.DONOTSAVECHANGES);
```


## 问题3: JS 里 bounds 返回 UnitValue 对象，导致位移计算错误（关键问题）
**现象**: 印花贴到了画布左上角外面，只有一小角露在 (0~92, 0~28)。
**原因**: 
1. JS 里 `bounds[0]` 返回的是 `UnitValue` 对象，直接参与运算会出错。
2. 没有设置标尺单位为像素，bounds 可能是其他单位。

**最终方案**:
```javascript
// 1. 设置单位为像素
app.preferences.rulerUnits = Units.PIXELS;

// 2. 用 .value 取数值
var layerLeft = bounds[0].value;
var layerTop = bounds[1].value;
```


## 问题4: 位移计算重复计算
**现象**: 在JSX里同时用了缩放后的bounds和缩放前的原始参数，导致重复计算。
**原因**: `resize()` 后 bounds 已经是缩放后的值，再用原始坐标算会出错。

**最终方案**: Python 传入缩放后的 `scaled_top_y` 和 `scaled_center_x`，JSX 里用缩放后的实际 bounds 计算：
```javascript
var moveX = targetCenterX - (layerLeft + scaledCenterX);
var moveY = targetTopY - (layerTop + scaledTopY);
```


## 问题5: 背图位置不精确
**现象**: 最初按比例估算的位置和用户手动摆放的不一致。
**解决**: 复刻美图秀秀的逻辑——
1. 计算贴图的"最高有效像素"y坐标
2. 计算贴图的"中心点"x坐标（所有有效像素x的平均值）
3. 缩放30%、旋转1°后，对齐到固定位置（X=680, Y=570）


## 问题6: JSX 路径转义问题
**现象**: 直接在 f-string 中写 `\\` 会被转义，导致 PS 找不到文件。
**解决**: 用 `.replace("\\", "\\\\")` 显式替换，确保 JSX 中路径是双反斜杠。


## 问题7: fitOnScreen() 报错
**现象**: `app.activeDocument.activeView.fitOnScreen()` 报错 "undefined 不是对象"。
**原因**: 该功能在某些版本的PS中不可用。
**解决**: 去掉这行代码（fitOnScreen 只是视图显示，不影响实际图像位置）。


## 问题8: BW 类型重复生成
**现象**: BW 类型的设计图，白T和黑T各生成了两次，前后覆盖。
**原因**: 最初的逻辑是 `if BW or W` 加 `if BW or B`，导致 BW 会走两个分支。
**解决**: 重构为 match-case 风格的任务列表，BW 单独处理，只生成一次。
