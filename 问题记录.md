# 开发过程中遇到的问题及解决方案

## 问题1: 透明PNG丢失透明度
**现象**: `selection.copy()` + `paste` 后印花变黑色矩形。
**原因**: copy+paste 丢失 alpha 通道。
**解决**: 改用 `ArtLayer.duplicate()` 图层复制，保留透明度。


## 问题2: JS 里 bounds 返回 UnitValue 对象
**现象**: 位置计算错误。
**解决**:
```javascript
app.preferences.rulerUnits = Units.PIXELS;
var value = bounds[0].value;  // 用 .value 取数值
```


## 问题3: 最高有效像素被噪点干扰
**现象**: Y方向位置偏差。
**解决**: 复刻美图秀秀——从上往下找，找到**一行有效像素≥50个**的那行。


## 问题4: 中心点x用简单平均值不准确
**现象**: X方向位置偏差。
**解决**: 用**透明度加权质心** `(x * alpha).sum() / alpha.sum()`。


## 问题5: 缩放比例基准不一致
**现象**: 正图"置入后设10%"比手动的小。
**解决**: 实测手动10%=相对原始13.33%，正图用 `scale_percent=13.33`。


## 问题6: PS resize(TOPLEFT) 锚点bug（最难发现）
**现象**: DX0086正确，DX0125偏下176像素。
**原因**: PS的 `resize(TOPLEFT)` 以**有效像素区域左上角**为锚点，不是图层(0,0)，透明区域不缩放。
**解决**: **Python预先trim透明边距 + 缩放**，保存为临时PNG，PS只负责duplicate+移动+旋转。


## 问题7: BW/WB分类逻辑错误
**现象**: DX0124_WB_cut.png 被误分类为W（只做正面，没做背面）。
**原因**: `_WB_` 不匹配 `_BW_`，但匹配了 `_W_`。
**解决**: `classify_design` 增加 `_WB_` 判断。


## 问题8: 符号链接跨域被浏览器阻止
**现象**: 点击图片打不开文件夹。
**原因**: HTML用 `file://` 打开，fetch请求 `http://localhost` 跨域。
**解决**: 改用本地HTTP服务，HTML也通过 `http://localhost:PORT/` 提供，同源访问。


## 问题9: thumbnail()返回None不能链式调用
**现象**: 缩略图生成失败。
**原因**: PIL的 `thumbnail()` 直接修改原对象，返回None。
**解决**: 拆成两行：`img.thumbnail(); img.save()`。


## 问题10: f-string中部分变量替换失败
**现象**: `{date_str}` 在生成的HTML中保留为字面值。
**原因**: 复杂f-string中某些`{var}`未被识别。
**解决**: 用占位符 `DATE_STR` 替代，最后 `.replace("DATE_STR", date_str)`。


## 问题11: 回收站恢复功能调不通
**现象**: PowerShell InvokeVerb("restore") 不生效。
**解决**: 改用 `Copy-Item -Path $item.Path -Destination "$orig\$name"` 直接复制回原位置。


## 问题12: 刷新后图片跑到其他日期（关键问题）
**现象**: 用户编辑图片后，文件mtime变更，刷新后图片从原日期消失。
**原因**: 刷新逻辑依赖文件mtime进行匹配。
**解决**: 引入 **`.dxlist` 清单文件**：
- 第一次生成时按mtime扫描，保存DX清单到 `.dxlist`
- **刷新时只用 `.dxlist`，不依赖文件mtime**
- 用户编辑图片后，图片仍留在原日期，不会跑丢
