# PS 正反图 — 问题与解决方法 v1.1.0

更新日期：2026-06-21

---

## 全部已修复

### 1. SaveAs 弹出 JPEG 选项对话框
**表现**：`doc.SaveAs(path, jpegOpt)` 打开保存弹窗，但需要手动点确定
**根因**：SaveAs 在未设置 DisplayDialogs 时会弹出 JPEG 选项对话框
**解决**：改用 **`doc.Export(ExportAs=2, Options=ExportOptionsSaveForWeb)`** 不弹窗直接保存
**验证**：Export 方式 Quality=100 等同于 SaveAs 的 Quality=12

### 2. DoJavaScript 报错"没有所需元素"（1302）
**表现**：`app.activeDocument` 返回 null
**根因**：PS 状态切换时，COM 和 ExtendScript 的活动文档不同步
**解决**：不再使用 DoJavaScript，改用 COM 的 Export 方法

### 3. PS "应用程序正忙" COM 调用阻塞
**表现**：设置 `ps.DisplayDialogs = 2` 时抛出 `-2147417846` 错误
**根因**：PS 有对话框打开时无法设置该属性
**解决**：不主动设置 DisplayDialogs，用无对话框的 Export 方式

### 4. 打开图片后需要等很久才能操作
**表现**：固定 sleep(4) + sleep(3) 等待图片加载
**解决**：改用 `ps_open_both()` 一次打开两张图 + `wait_ps_docs(2)` 轮询等待文档就绪
**优化效果**：从固定 7s 等待优化为实际加载时间（通常 3-4s）

### 5. 打开顺序导致切标签
**表现**：背和正都打开后，需要手动激活背的标签
**根因**：PS 最后打开的文件自动激活
**解决**：命令行传参顺序改为 **正先 → 背后**，背后打开就是活动文档

### 6. PS 已有的垃圾标签页干扰
**表现**：PS 已有其他图片打开时，新打开的标签混在一起
**解决**：打开新图前先调用 `close_all_docs(ps)` 清空所有标签
