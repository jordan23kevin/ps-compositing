# 开发过程中遇到的问题及解决方案

## 问题1: 透明PNG丢失透明度（关键问题）
**现象**: 用 `selection.copy()` + `paste` 粘贴透明 PNG 后，印花变成黑色矩形，盖住胚衣。
**原因**: `selection.selectAll()` + `copy` + `paste` 的方式会丢失 alpha 通道，透明区域变成黑色。

**最终方案**: 改用 `ArtLayer.duplicate()` 图层复制方式，完整保留透明度：
```javascript
var designLayer = designDoc.artLayers[0].duplicate(doc);
designDoc.close(SaveOptions.DONOTSAVECHANGES);
```


## 问题2: JS 里 bounds 返回 UnitValue 对象（关键问题）
**现象**: 印花贴到了画布左上角外面。
**原因**: 
1. JS 里 `bounds[0]` 返回的是 `UnitValue` 对象，直接参与运算会出错。
2. 没有设置标尺单位为像素，bounds 可能是其他单位。

**最终方案**:
```javascript
app.preferences.rulerUnits = Units.PIXELS;
var layerLeft = bounds[0].value;  // 用 .value 取数值
var layerTop = bounds[1].value;
```


## 问题3: 最高有效像素计算逻辑错误
**现象**: 位置有偏差。
**原因**: 直接取 `ys.min()` 作为最高有效像素，会被顶部噪点干扰。
**最终方案**: 复刻美图秀秀 `calc_offset_back` 逻辑——从上往下找，找到**一行有效像素≥50个**的那行才算：
```python
y_min = int(ys.min())
for y in range(y_min, min(y_min + 200, h)):
    if (alpha[y] >= ALPHA_THRESHOLD).sum() >= 50:
        y_min = y
        break
```


## 问题4: 中心点x计算错误
**现象**: 位置x有偏差。
**原因**: 用简单平均值 `xs.mean()`，但美图秀秀用的是**透明度加权质心**。
**最终方案**:
```python
weights = np.where(mask, alpha, 0.0)
center_x = float((np.indices(alpha.shape)[1] * weights).sum() / weights.sum())
```


## 问题5: 缩放比例基准不一致（关键问题，最难发现）
**现象**: 正图用 resize(10) 生成，但用户说比手动"置入后设10%"小，目测只有7.5%。
**原因**: 
- 用户手动操作是"PS置入 + 设10%"，PS置入时会自动缩放
- 我用 duplicate（不经过置入），resize(10) 是相对原始尺寸2048的10%
- 两者基准不同，结果不同

**调试过程**:
1. 用户测量手动结果：整个贴图273px宽，有效区域155×234
2. 反推：273/2048 = 13.33%，说明手动"10%"实际等效于相对原始尺寸13.33%
3. 用 duplicate + resize(13.33) 测试，生成有效区域153×234，与手动155×234吻合

**最终方案**: 正图用 `scale_percent = 13.33`（不是10），背图用 `scale_percent = 30`。
两者都用 duplicate 方式，缩放基准统一为原始尺寸2048。


## 问题6: 用错误的Python差异检测验证位置（教训）
**现象**: 我用 `diff_sum > 10` 检测贴图位置，显示Y=600、X=843，然后错误地说"接近✅"。
**原因**: 旋转1°后，贴图边角的半透明像素被算进差异区域，导致检测位置偏大、偏左。这种检测方法不能准确反映贴图真实位置。
**教训**: 不要用不准确的检测方法下结论。应该直接在JSX里读取图层实际bounds来验证。


## 问题7: JSX 路径转义问题
**现象**: 直接在 f-string 中写 `\\` 会被转义。
**解决**: 用 `.replace("\\", "\\\\")` 显式替换。


## 问题8: fitOnScreen() 报错
**现象**: `app.activeDocument.activeView.fitOnScreen()` 报错。
**解决**: 去掉这行代码（只是视图显示，不影响实际图像位置）。


## 问题9: BW 类型重复生成
**现象**: BW 类型的设计图，白T和黑T各生成了两次。
**原因**: 最初的逻辑 `if BW or W` 加 `if BW or B`，导致 BW 走两个分支。
**解决**: 重构为独立分支，BW 单独处理。
