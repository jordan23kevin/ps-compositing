# WB 贴图 Photoshop 版 — 项目架构

## 概述
将原本美图秀秀的 WB 贴图流程迁移到 Photoshop，使用 JSX + Python win32com 自动化实现。

```

```

## 核心算法

### 贴图位置计算（calculate_sticker_position）

复刻美图秀秀 `calc_offset_back` 逻辑：
1. **最高有效像素 y**：alpha >= 20 的像素中，从上往下找到**一行有效像素≥50个**的那行
   （防止顶部噪点干扰）
2. **中心点 x**：透明度加权质心 `(x * alpha).sum() / alpha.sum()`

### Python预处理（关键！）

**Python预先trim透明边距 + 缩放贴图**，保存为临时PNG：
1. 读取贴图为RGBA
2. 用 alpha >= 20 做掩码，找到有效像素区域
3. **trim透明边距**：crop到有效像素外接框 `(x0,y0)~(x1,y1)`
4. **缩放**：按配置的scale_percent缩放（相对于原始尺寸）
5. 保存为临时PNG

> **为什么Python预处理？** PS的 `resize(TOPLEFT)` 锚点有bug——
> 它以有效像素区域左上角为锚点，而不是图层(0,0)，
> 导致透明区域不缩放，bounds.T不正确，位置偏移。
> Python预处理完全避开这个bug。

### PS操作（place_design.jsx）

1. 打开胚衣
2. 打开Python预处理后的临时PNG
3. `ArtLayer.duplicate()` 复制图层（保留透明度）
4. `translate(moveX, moveY)` 移动到目标位置
5. `rotate(rotation)` 旋转
6. 保存JPG

> trim后图层左上角=有效像素左上角，所以：
> - moveX = target_center_x - new_w/2（中心对齐）
> - moveY = target_top_y（顶部对齐）

## 关键参数（config.py）

### 背图 (BACK_NEW)
```python
BACK_NEW = {
    "torso_white": "白背2.jpg",
    "torso_black": "黑背2.jpg",
    "scale_percent": 30,      # 缩放30%（相对原始2048）
    "rotation": 1,            # 向右旋转1°
    "target_center_x": 680,   # 贴图中心点目标X
    "target_top_y": 570,      # 贴图最高有效像素目标Y
}
```

### 正图 (FRONT_NEW)
```python
FRONT_NEW = {
    "torso_white": "白正2.jpg",
    "torso_black": "黑正2.jpg",
    "scale_percent": 13.33,    # 实测等效于手动置入后设10%
    "rotation": 1,            # 向右旋转1°
    "target_center_x": 888,   # 贴图中心点目标X
    "target_top_y": 612,      # 贴图最高有效像素目标Y
}
```

> **关于正图13.33%**：用户手动操作是"置入贴图后设10%"。
> 实测手动"10%"等效于相对原始尺寸(2048)的13.33%。
> Python预处理直接用13.33%即可复刻手动效果。
> 验证：生成贴图有效区域 153×234px，手动测量 155×234px，吻合。

## 两阶段处理流程

### 第一阶段：贴图（wb_sticker_ps.py）

1. 扫描 `SOURCE_BASE` 下所有 `DX*` 文件夹
2. 从 `DX*/02_REM_BG/` 读取 `_cut.png`
3. 根据文件名分类：BW / W / B
4. 生成贴图：
   - **B类型**: 背图，生成 `DXxxxx_B_白T.jpg`，`DXxxxx_B_黑T.jpg`
   - **W类型**: 正图，生成 `DXxxxx_W_白T.jpg`，`DXxxxx_W_黑T.jpg`
   - **BW类型**: 同时生成正面和背面共4个文件（供第二阶段合成）

### 第二阶段：BW合成（ps_batch.py，仅BW类型需要）

1. 从 `DX*/03_UPLOAD/` 读取 `_W_*T.jpg` 和 `_B_*T.jpg`
2. 调用 Photoshop 的 `正反图` 动作组
3. 生成 `DXxxxx_白BW.jpg` 和 `DXxxxx_黑BW.jpg`
