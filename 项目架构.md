# WB 贴图流水线 — 项目架构（纯软件，v2.5.0）

> 完整跨仓库架构 / 方法 / 问题方案 / 复现回滚见 **`PIPELINE.md`**（本目录）。
> 历史 PS 版（JSX + win32com）架构已废弃，仅保留在 git 历史备查。

## 概述
本仓库负责**平铺图**（衣服摊平拍、无褶皱）的贴花与 BW 合成。自 v2.0.0 起**完全不依赖 Photoshop**，全部用 PIL 纯软件实现。模特图（人穿着、有褶皱）由 `white_t_mockup` 仓库处理（见 PIPELINE.md 第四章）。

## 项目结构
```
E:\Claude code\ps\
├── config.py            # 胚衣路径、FLAT_TORSO 映射、.meta.json 五参加载；FRONT_NEW/BACK_NEW 仅旧线回退
├── wb_sticker_ps.py     # 平铺图贴花 StickerSession（纯 PIL affine），v2.5.1（五参定位；黑衫默认不加白墨打底，按用户批准效果）
├── ps_batch.py          # BW 合成 compose_bw_pil（纯 PIL），v2.0.0
├── ps_sticker_one.py    # 通用平铺贴花入口（subprocess 被 check_rem 调用）
├── process_black.py     # 黑T 专用贴花 + BW 合成，v2.5.0（纯软件）
├── process_white.py     # 白T 专用贴花 + BW 合成，v2.5.0（纯软件）
├── ps_batch_one.py      # 单款 BW 合成入口（不连 PS）
├── wb_meta.py           # UID/group_id 元数据注册
├── quit_ps.py           # 退出 PS（检测 PS 未运行则直接返回，安全无副作用）
├── PIPELINE.md          # 跨仓库主文档（架构/方法/问题/复现/回滚）
├── CHANGELOG.md / 运行手册.md / 问题记录.md / 项目架构.md
└── jsx/                 # 遗留：旧 PS 版 JSX（v2.0.0 起不再使用，保留备查）
```

## 处理流程（纯软件）
1. **平铺贴花** `wb_sticker_ps.StickerSession.place_design`
   - trim 透明边距 → 读取 `.meta.json` 五参 → 按 width/设计宽缩放 → 以「顶边中点」为参考点平移旋转 → normal 合成。
   - 定位参数优先来自 `D:\Semems WB\03_MATERIAL` 各胚衣同名 `.meta.json`；`config.FRONT_NEW / BACK_NEW` 仅作为 `process_black.py/process_white.py` 旧线回退。
2. **BW 合成** `ps_batch.compose_bw_pil`
   - 底图 1340×1785，圆直径 595，正面图宽度贴合圆圈（≈44.4%）垂直居中，圆心 (1014,1449)，白边 5px，无阴影。
   - 单面款自动清理残留 BW。

## 与 orchestrator 的关系
`check_rem.py`（ZCodeProject，端口 8766 常驻）「📎 贴图」按钮按序 subprocess 调用 `process_black → process_white → ps_sticker_one → ps_batch_one`，全程纯软件，**不再打开 Photoshop**。`quit_ps.py` 在末尾被调用，但检测 PS 未运行即返回。
