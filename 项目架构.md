## 3. 核心脚本

### `ps_batch.py` — 主批处理 v1.1.0

**流程：**
1. 扫描 `D:\Semems\1AI` 下所有 DX 文件夹
2. 每文件夹先处理白T → 再处理黑T
3. 跳过已存在 `白BW.jpg` / `黑BW.jpg` 的文件夹

**单次处理（process_color）：**
1. 连接 PS → 关闭所有已有标签（保证只有需要的两张图）
2. `ps_open_both(正, 背)` → 一次命令行打开正背，**背最后打开自动激活**
3. `wait_ps_docs(2)` → 轮询等待 2 个文档打开（0.5s 间隔，不用固定 sleep）
4. `DoAction('白'/'黑', '正反图')` → 运行动作
5. `Export(ExportOptionsSaveForWeb)` → 保存 JPEG 品质12（不弹窗）
6. `Close(2)` → 关闭所有文档

## 4. 关键实现细节

### 4.1 保存方式（踩坑记录）

| 方式 | 结果 |
|------|------|
| `doc.SaveAs(jpegOpt)` | ⛔ 弹出 JPEG 选项对话框，需要手动点确定 |
| `SaveForWeb` COM | ⛔ 属性设置不兼容 |
| `DoJavaScript(doc.saveAs)` | ⛔ 有时活动文档为 null，报错 1302 |
| **`Export(ExportOptionsSaveForWeb)`** | ✅ **不弹窗，直接保存** |

**最终方案：** `doc.Export(ExportIn=path, ExportAs=2, Options=export_opts)`
- `ExportAs=2` = SaveForWeb 格式
- `Format=6` = JPEG
- `Quality=100` = Photoshop 最高品质 12

### 4.2 打开顺序（关键）

PS 命令行打开多个文件时，**最后打开的文件自动成为活动文档**。

```
正确顺序：命令行传参 [正] [背] → 正先打开 → 背后打开并激活
```

这样不需要额外用 API 切标签，背打开后直接可以运行动作。

### 4.3 速度优化

- **一次性打开两张图**：一次命令行传两个文件，PS 同时开启
- **COM 轮询代替固定 sleep**：`wait_ps_docs(2)` 每 0.5s 检查文档数
- 从原来的固定 7s 等待优化为实际加载时间（通常 3-4s）

## 7. 注意事项

1. PS 动作 `正反图 > 白` 和 `正反图 > 黑` 必须在动作面板中存在
2. 文件夹中必须同时有 `X背2_副本.jpg` 才能处理（只有正没有背会跳过）
3. 保存品质 100 = Photoshop 的 JPEG 最高品质 12
4. `DisplayDialogs` 设置可能导致 COM 调用阻塞（PS 繁忙），所以不主动设置
