## 两阶段处理流程

### 第一阶段：贴图（wb_sticker_ps.py）
```bash
python wb_sticker_ps.py [起始DX]
```
1. 从 `02_REM_BG/` 读取 `_cut.png`
2. 通过 **Python 预处理**（trim透明边距 + 缩放）保存为临时PNG
3. PS 执行 `duplicate → move → rotate`
4. 生成到 `03_UPLOAD/`
5. 自动计时：每张、每款、总耗时

### 第二阶段：BW合成（ps_batch.py）
```bash
python ps_batch.py [起始DX]
```
1. 从 `03_UPLOAD/` 读取 `_W_*T.jpg` 和 `_B_*T.jpg`
2. 调用 Photoshop 的 `正反图` 动作组
3. 生成 `DXxxxx_白BW.jpg` 和 `DXxxxx_黑BW.jpg`

## 前端预览服务（check_sync.py）

本地HTTP服务（端口8765），功能：

| 功能 | 触发方式 | 说明 |
|------|---------|------|
| 图片预览 | 浏览器打开 | 按日期分组显示黑BW缩略图 |
| 打开文件夹 | 点击图片 | 资源管理器打开对应03_UPLOAD |
| 同步删除 | 点击按钮 | 删缩略图→对应原图送回收站 |
| 恢复 | 点击按钮 | 从回收站还原已删文件 |
| 刷新 | 点击按钮 | 根据.dxlist重新生成缩略图（不受文件mtime影响） |
| 切换日期 | 下拉框 | 切换不同日期的预览 |
| 单张删除 | ×按钮 | 单张缩略图送回收站 |

### 核心机制：.dxlist 文件
- 第一次生成页面时按文件mtime扫描，保存DX清单到 `.dxlist`
- **刷新时只用 `.dxlist`，不依赖文件mtime**
- 用户编辑图片后，图片仍留在原日期，不会跑丢

## 关键参数（config.py）

### 背图 (BACK_NEW)
```python
BACK_NEW = {
    "scale_percent": 30,      # 缩放30%
    "rotation": 1,            # 向右旋转1°
    "target_center_x": 680,   # 中心点类型X
    "target_top_y": 570,      # 最高有效像素类型Y
}
```

### 正图 (FRONT_NEW)
```python
FRONT_NEW = {
    "scale_percent": 13.33,    # 等效手动置入后设10%
    "rotation": 1,
    "target_center_x": 888,
    "target_top_y": 612,
}
```
