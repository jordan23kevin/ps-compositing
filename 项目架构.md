# WB 贴图 Photoshop 版 — 项目架构

## 概述
将原本美图秀秀的 WB 贴图流程迁移到 Photoshop，使用 JSX + Python win32com 自动化实现。

## 核心解决的问题
1. **设计图透明背景**: 设计图是 2048×2048 PNG，**已有透明 alpha 通道**（alpha>=20 为有效像素），不是黑色背景。

2. **精确位置控制**: 复刻美图秀秀的逻辑——计算贴图的"最高有效像素"和"中心点"，对齐到 PS 坐标系的固定位置。

3. **保留透明度**: JS 里用 `selection.copy()` + `paste` 会丢失 alpha 通道，必须用 `ArtLayer.duplicate()` 图层复制。

4. **单位问题**: JS 里 `bounds` 返回 UnitValue 对象，必须设置 `rulerUnits = PIXELS` 并用 `.value` 取数值。

5. **BW 类型两阶段处理**: 
   - 第一阶段：用新脚本生成正面和背面贴图
   - 第二阶段：用原来的 `ps_batch.py` 调用 Photoshop 动作合成最终 BW 图

## 关键参数（config.py）

### 背图新方案 (BACK_NEW)
```python
BACK_NEW = {
    "torso_white": "白背2.jpg",
    "torso_black": "黑背2.jpg",
    "scale_percent": 30,      # 缩放30%
    "rotation": 1,            # 向右旋转1°
    "target_center_x": 680,   # 贴图中心点目标X
    "target_top_y": 570,      # 贴图最高有效像素目标Y
}
```

## 背图新方案处理流程

### 第一步：Python 计算贴图位置（calc_position.py / wb_sticker_ps.py）
1. 读取贴图为 RGBA
2. 用 alpha >= 20 做掩码
3. **最高有效像素 y** = `np.where(mask)[0].min()`（从上往下第一个有效像素的y）
4. **中心点 x** = 所有有效像素 x 的平均值 `xs.mean()`
5. 缩放30%后：`scaled_top_y = top_y * 0.3`，`scaled_center_x = center_x * 0.3`

### 第二步：JSX 执行 PS 操作（jsx/place_back_new.jsx）
1. 设置 `rulerUnits = PIXELS`
2. 打开胚衣
3. 打开设计图，用 `ArtLayer.duplicate()` 复制图层到胚衣文档（保留透明度）
4. 关闭设计图文档
5. 缩放 30%（`AnchorPosition.TOPLEFT`）
6. 旋转 1°（向右）
7. 读取缩放后的图层 bounds，用 `.value` 取数值
8. 计算位移：
   - `moveX = target_center_x - (layerLeft + scaled_center_x)`
   - `moveY = target_top_y - (layerTop + scaled_top_y)`
9. `designLayer.translate(moveX, moveY)` 移动到目标位置
10. 保存 JPG

## 两阶段处理流程

### 第一阶段：贴图（wb_sticker_ps.py）

1. **扫描 DX 文件夹**: 遍历 `SOURCE_BASE` 下所有 `DX*` 文件夹
2. **读取设计图**: 从 `DX*/02_REM_BG/` 读取 PNG
3. **分类**: 根据文件名判断是 BW / W / B
4. **生成贴图**:
   - **B类型**: 用新方案背图，生成 `DXxxxx_B_白T.jpg`，`DXxxxx_B_黑T.jpg`
   - **W类型**: 待正图新方案完成后补充
   - **BW类型**: 生成背面文件，正面待补充（供第二阶段合成）

### 第二阶段：BW合成（ps_batch.py，仅BW类型需要）

1. **读取贴图结果**: 从 `DX*/03_UPLOAD/` 读取 `_W_*T.jpg` 和 `_B_*T.jpg`
2. **调用PS动作**: 使用 Photoshop 中预先定义的 `正反图` 动作组
3. **生成最终BW图**: 输出 `DXxxxx_白BW.jpg` 和 `DXxxxx_黑BW.jpg`
