# WB 贴图 Photoshop 版 — 项目架构

## 概述
将原本美图秀秀的 WB 贴图流程迁移到 Photoshop，使用 JSX + Python win32com 自动化实现。

## 核心解决的问题
1. **设计图黑色背景处理**: 原始设计图是 2048×2048 PNG，背景为纯黑，而不是透明 alpha。
   - 解决方法: 使用 Python PIL 预先将黑色像素 (RGB<20) 转为透明，保存为临时 PNG，再在 PS 中使用。

2. **精确位置控制**: 完全复刻用户手动在 PS 中摆放的位置参数。

3. **BW 类型两阶段处理**: 
   - 第一阶段：用新脚本生成正面和背面贴图
   - 第二阶段：用原来的 `ps_batch.py` 调用 Photoshop 动作合成最终 BW 图

## 文件结构

```
ps/
├── config.py                 # 配置文件（路径、坐标、缩放、旋转）
├── wb_sticker_ps.py          # 主程序入口（第一阶段：贴图）
├── ps_batch.py               # 原来的合成脚本（第二阶段：BW合成）
├── jsx/
│   ├── place_front.jsx       # 正面贴图 JSX 脚本
│   └── place_back.jsx        # 背面贴图 JSX 脚本
├── 运行手册.md               # 使用说明
├── 问题记录.md               # 开发过程中遇到的问题及解决方案
└── 项目架构.md               # 本文档
```

## 关键参数（config.py）

### 背图参数 (BACK)
- target_height: 498 px (约为原图的 29%)
- center_x: 718
- center_y: 808
- rotation: 0°

### 正图参数 (FRONT)
- target_height: 400 px
- center_x: 850
- center_y: 600
- rotation: 0°

## 两阶段处理流程

### 第一阶段：贴图（wb_sticker_ps.py）

1. **扫描 DX 文件夹**: 遍历 `SOURCE_BASE` 下所有 `DX*` 文件夹
2. **读取设计图**: 从 `DX*/02_REM_BG/` 读取 PNG
3. **分类**: 根据文件名判断是 BW / W / B
4. **预处理背景**: 使用 PIL 将黑色背景转为透明
5. **生成贴图**:
   - **B类型**: 只生成背面贴图（`DXxxxx_B_白T.jpg`，`DXxxxx_B_黑T.jpg`）
   - **W类型**: 只生成正面贴图（`DXxxxx_W_白T.jpg`，`DXxxxx_W_黑T.jpg`）
   - **BW类型**: 同时生成正面和背面共4个文件（为第二阶段合成做准备）

### 第二阶段：BW合成（ps_batch.py，仅BW类型需要）

1. **读取贴图结果**: 从 `DX*/03_UPLOAD/` 读取 `_W_*T.jpg` 和 `_B_*T.jpg`
2. **调用PS动作**: 使用 Photoshop 中预先定义的 `正反图` 动作组
3. **生成最终BW图**: 输出 `DXxxxx_白BW.jpg` 和 `DXxxxx_黑BW.jpg`

## 依赖
- Python 3.x
- pywin32 (win32com.client)
- Pillow (PIL)
- Photoshop (已安装并能正常启动，且包含 `正反图` 动作组)
